# 📊 Telecom-Churn-Analysis-in-Power-BI
A Power BI case study analysing customer churn for **Databel**, a fictitious telecom company.  
Built using **Power BI**, **Power Query**, and **DAX**.

---

## 📌 Project Overview

This project investigates *why* customers are leaving Databel and surfaces actionable insights to reduce churn. It was completed as part of a structured Power BI learning journey.

## 🔍 Key Insights

| # | Insight |
|---|---------|
| 📉 | **26.86%** of customers churned |
| 🏆 | Leading churn causes: competitors offering better products/services |
| 📅 | Month-to-month contracts account for **51%** of all churners |
| 📶 | Unlimited plan customers using **less than 5GB** had the highest churn rate |
| 👴 | Churn was highest among customers **aged 65+** |
| 📞 | Customers who made **customer service calls** were significantly more likely to churn |
| 💳 | **55%** of churned customers used **direct debit** as their payment method |

---

## ✅ Recommendations for Databel

1. **Strengthen competitive position** — improve products, pricing, and service offerings
2. **Retain month-to-month customers** — offer discounts, loyalty rewards, and contract upgrade incentives
3. **Recommend better-fit plans** — identify customers underutilising their data and offer right-sized plans
4. **Senior customer retention strategy** — simpler plans, proactive support, and more human touchpoints
5. **Improve customer service** — reduce resolution times, improve first-call resolution rates
6. **Investigate direct debit churn** — review billing processes, improve transparency, offer flexible payment options
